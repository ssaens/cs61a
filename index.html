List Operations

Python lists are ordered collections of objects. There are many common python operations that may be used to combine and modify lists. Some will mutate our data structures and others make new lists. It's important to understand what list functions are doing to your data structures to ensure that functions you write are doing exactly what you want them to do without unintended side effects.

Here is a list of common operations:

Iterators, Generators, and Streams

Iterators

Iterators are objects which return the elements of some sequence in some order. They must implement the __next__ function. When called, __next__ returns the next element of the underlying sequence it represents. Calling __next__ mutates the iterator, advancing the position of the iterator. An iterator will raise StopIteration exception upon reaching the end of its sequence.

Iterables

Iterables represent collections of data which may be traversed in some order. Iterables contain a __iter__ method, which, once called, returns an iterator. Every call to __iter__ returns a new iterator object. If a class has both the __iter__ and __next__ methods, it is both and iterable, and iterator.

Generators

Generators are an alternate method to iterate through a sequence. Generators are created through generator functions which use a yield statement rather than a return. When a generator function is called, it returns a generator object. Calling next on a generator will execute the body of the generator function that created with until a yield statement is reached. The yielded value is then the return value of the next call. Upon calling next again, the generator function body continues execution where it left off until it reaches another yield or returns a value (remember that if a function doesn't explictly have a return value, it returns None), raising a StopIteration exception.
